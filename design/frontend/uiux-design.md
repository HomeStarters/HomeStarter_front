# UI/UX 설계서

## 1. UI 프레임워크 선택

### 선택: Material-UI (MUI)
**선택 이유:**
- React 18.3과 완벽한 호환성
- 모바일 우선 반응형 디자인 지원
- 금융 서비스에 적합한 전문적인 디자인 시스템
- 다양한 컴포넌트와 커스터마이징 용이성
- 접근성(a11y) 표준 준수
- TypeScript 네이티브 지원

## 3. 화면 간 사용자 플로우

```
[로그인] → [대시보드]
           ↓
     ┌─────┴─────┐
     ↓           ↓
[프로필/자산]  [주택관리]
     ↓           ↓
[기본정보입력] [주택등록]
     ↓           ↓
[자산입력]    [계산하기]
     ↓           ↓
[자산관리]    [계산결과]
                 ↓
            [로드맵조회]
```

### 주요 사용자 여정

#### 여정 1: 신규 사용자 온보딩
1. 회원가입 → 로그인
2. 기본정보 입력 (주소, 직장, 투자성향)
3. 본인 자산정보 입력
4. 배우자 자산정보 입력 (선택)
5. 대시보드에서 재무 현황 확인

#### 여정 2: 주택 탐색 및 계산
1. 대시보드
2. 주택 등록 (기본정보 → 상세정보)
3. 대출상품 조회
4. 지출 계산
5. 계산결과 확인
6. 로드맵 생성/조회

#### 여정 3: 관리자 운영
1. 로그인 (관리자)
2. 대출상품 관리
3. 상품 등록/수정/삭제

## 4. 화면별 상세 설계

### 4.1 로그인 화면 (`/login`)

**상세기능:**
- 아이디/비밀번호 입력
- 로그인 유지 옵션
- 로그인 처리
- 회원가입 링크

**UI 구성요소:**
- AppBar: Back 버튼, "로그인" 타이틀
- TextField: 아이디 입력
- TextField: 비밀번호 입력 (password type)
- Checkbox: 로그인 유지
- Button: 로그인 버튼 (primary)
- Link: 회원가입 링크

**인터랙션:**
- 입력 검증: 필수 필드 체크
- 로그인 성공: JWT 토큰 저장 → 대시보드 이동
- 로그인 실패: 에러 메시지 표시 (Snackbar)

### 4.2 회원가입 화면 (`/register`)

**상세기능:**
- 개인정보 입력 (이름, 이메일, 전화번호)
- 계정정보 입력 (아이디, 비밀번호, 비밀번호 확인)
- 약관 동의
- 회원가입 처리

**UI 구성요소:**
- AppBar: Back 버튼, "회원가입" 타이틀
- TextField: 이름 (2-50자)
- TextField: 이메일 (이메일 형식)
- TextField: 전화번호 (01X-XXXX-XXXX)
- TextField: 아이디 (5-20자, 영문+숫자)
- TextField: 비밀번호 (8자 이상)
- TextField: 비밀번호 확인
- Checkbox: 약관 동의
- Button: 회원가입 버튼

**인터랙션:**
- 실시간 입력 검증
- 비밀번호 일치 확인
- 약관 동의 필수
- 성공: 로그인 페이지 이동
- 실패: 에러 메시지 표시

### 4.3 대시보드 (`/dashboard`)

**상세기능:**
- 사용자 재무 현황 요약
- 빠른 작업 메뉴
- 최근 활동 표시

**UI 구성요소:**
- AppBar: 사용자 프로필, 알림 아이콘
- Card: 재무 현황 (총 자산, 총 대출, 순자산, 월 가용자금)
- Grid: 빠른 작업 카드
  - 기본정보 및 자산 등록
  - 주택 등록하기
  - 지출 계산하기
  - 로드맵 보기
  - 대출상품 관리 (관리자)
- Tabs: 최근 활동 (등록한 주택 / 계산 결과)
- BottomNavigation: 주요 메뉴 (대시보드, 주택, 계산, 로드맵)

**인터랙션:**
- 재무 현황 클릭: 자산정보 관리 페이지 이동
- 빠른 작업 클릭: 해당 기능 페이지 이동
- 프로필 클릭: 프로필 편집 페이지 이동

### 4.4 프로필 편집 (`/profile/edit`)

**상세기능:**
- 사용자 기본정보 표시
- 프로필 정보 수정

**UI 구성요소:**
- AppBar: Back 버튼, "프로필 편집" 타이틀
- Avatar: 사용자 이니셜
- TextField: 이름 (읽기 전용)
- TextField: 이메일 (읽기 전용)
- TextField: 전화번호
- TextField: 생년월일 (읽기 전용)
- Select: 성별 (읽기 전용)
- Button: 저장 버튼

**인터랙션:**
- 수정 가능 필드만 활성화
- 저장 버튼 클릭: API 호출 → 성공 메시지 → 대시보드 이동

### 4.5 기본정보 입력 (`/profile/basic-info`)

**상세기능:**
- 현재 주소 입력
- 본인 직장 주소 입력
- 배우자 직장 주소 입력
- 투자 성향 선택

**UI 구성요소:**
- AppBar: Back 버튼, "기본정보 입력" 타이틀
- Stepper: 진행 단계 표시
- AddressInput (커스텀 컴포넌트):
  - TextField: 도로명 주소
  - TextField: 지번 주소
  - TextField: 우편번호
  - TextField: 위도/경도
- RadioGroup: 투자 성향 (HIGH, MEDIUM, LOW)
- Button: 이전, 다음 버튼

**인터랙션:**
- 주소 검색 기능 (카카오 주소 API)
- 단계별 진행
- 저장 버튼: API 호출 → 자산입력 페이지 이동

### 4.6 본인 자산정보 입력 (`/assets/self`)

**상세기능:**
- 자산 항목 추가/삭제
- 대출 항목 추가/삭제
- 월 소득 항목 추가/삭제
- 월 지출 항목 추가/삭제
- 총계 자동 계산

**UI 구성요소:**
- AppBar: Back 버튼, "본인 자산정보 입력" 타이틀
- Accordion: 자산, 대출, 월소득, 월지출
- List + ListItem: 각 항목
  - TextField: 항목명
  - TextField: 금액 (숫자, 단위: 원)
  - IconButton: 삭제 버튼
- Button: 항목 추가 버튼
- Card: 총계 표시
- Button: 저장 버튼

**인터랙션:**
- 항목 추가: 새 입력 필드 추가
- 항목 삭제: 확인 후 삭제
- 금액 입력: 숫자만 허용, 천단위 콤마 자동
- 저장: API 호출 → 성공 메시지 → 대시보드 이동

### 4.7 배우자 자산정보 입력 (`/assets/spouse`)

**상세기능:**
- 배우자 없음 체크박스
- 자산 항목 추가/삭제
- 대출 항목 추가/삭제
- 월 소득 항목 추가/삭제
- 월 지출 항목 추가/삭제

**UI 구성요소:**
- AppBar: Back 버튼, "배우자 자산정보 입력" 타이틀
- Checkbox: 배우자 없음
- (본인 자산정보와 동일한 구조)

**인터랙션:**
- 배우자 없음 체크: 모든 입력 비활성화, 값 0으로 설정
- 기타 인터랙션은 본인 자산정보와 동일

### 4.8 자산정보 관리 (`/assets`)

**상세기능:**
- 본인/배우자 자산 조회
- 통합 요약 정보
- 자산정보 수정

**UI 구성요소:**
- AppBar: Back 버튼, "자산정보 관리" 타이틀
- Tabs: 본인 / 배우자 / 통합
- Card: 총 자산, 총 대출, 순자산, 월 가용자금
- Accordion: 자산, 대출, 월소득, 월지출 상세
- Button: 수정 버튼

**인터랙션:**
- 탭 전환: 본인/배우자 데이터 전환
- 수정 버튼: 해당 자산입력 페이지 이동
- 통합 탭: 본인+배우자 합산 표시

### 4.9 대출상품 목록 (`/loans`)

**상세기능:**
- 대출상품 목록 조회
- 필터링 (주택유형)
- 정렬 (금리, 한도, 등록일)
- 검색 (상품명, 대상주택)
- 페이징

**UI 구성요소:**
- AppBar: Back 버튼, "대출상품" 타이틀, 검색 아이콘
- TextField: 검색 입력
- Chip: 주택유형 필터
- Select: 정렬 기준
- Card List: 대출상품 카드
  - 상품명
  - 금리, 한도
  - LTV/DTI/DSR 한도
  - 대상 주택
- Pagination: 페이지 네비게이션

**인터랙션:**
- 검색: 입력 후 Enter 또는 검색 버튼
- 필터 선택: 자동 검색
- 정렬 변경: 자동 재정렬
- 카드 클릭: 상세 페이지 이동
- 페이지 변경: 다음/이전 페이지 로드

### 4.10 대출상품 상세 (`/loans/:id`)

**상세기능:**
- 대출상품 상세정보 조회
- 대출 조건 표시

**UI 구성요소:**
- AppBar: Back 버튼, "대출상품 상세" 타이틀
- Card: 상품 정보
  - Typography: 상품명
  - Divider
  - List: 상세 정보
    - 대출 한도
    - LTV/DTI/DSR 한도
    - 금리
    - 대상 주택
    - 소득 요건
    - 신청자격
    - 비고

**인터랙션:**
- Back 버튼: 목록으로 이동

### 4.11 대출상품 관리 (`/admin/loans`)

**상세기능:**
- 대출상품 목록 조회 (관리자)
- 상품 등록/수정/삭제
- 활성화/비활성화 관리

**UI 구성요소:**
- AppBar: Back 버튼, "대출상품 관리" 타이틀, 추가 버튼
- Table: 대출상품 리스트
  - 상품명, 금리, 한도, 상태
  - IconButton: 수정, 삭제
- Dialog: 상품 등록/수정 폼
- Switch: 활성화 토글

**인터랙션:**
- 추가 버튼: 등록 다이얼로그 열기
- 수정 버튼: 수정 다이얼로그 열기
- 삭제 버튼: 확인 다이얼로그 → API 호출
- 저장: API 호출 → 성공 메시지 → 목록 갱신

### 4.12 입주희망주택 입력 - 기본정보 (`/housings/new/basic`)

**상세기능:**

- Stepper: 1/2 단계
- TextField: 주택명
- RadioGroup: 주택유형 (아파트, 오피스텔, 빌라, 단독주택)
- TextField: 가격 (숫자)
- TextField: 입주희망년월 (YYYY-MM)
- AddressInput: 주소
- Button: 다음 버튼

**인터랙션:**
- 필수 입력 검증
- 다음 버튼: 상세정보 페이지 이동 (데이터 전달)

### 4.13 입주희망주택 입력 - 상세정보 (`/housings/new/detail`)

**상세기능:**
- 단지 정보 입력
- 교통호재 입력
- 생활환경 입력
- 주택 저장

**UI 구성요소:**
- AppBar: Back 버튼, "주택 등록" 타이틀
- Stepper: 2/2 단계
- Accordion: 단지정보
  - TextField: 단지명, 총 세대수, 총 동수, 총 층수
  - TextField: 주차 대수, 입주일, 시공사
  - TextField: 전용면적, 공급면적, 층, 향
- Accordion: 교통호재
  - Select: 교통수단 (지하철, 버스, 기차)
  - TextField: 노선명, 역/정류장명
  - TextField: 거리(m), 도보시간(분)
  - TextField: 출퇴근 시간
  - Button: 추가
- Accordion: 생활환경
  - RadioGroup: 일조량, 소음 수준
  - Chip Input: 인근 학교, 마트, 병원
- Button: 이전, 저장 버튼

**인터랙션:**
- 교통호재 추가: 리스트에 추가
- 저장 버튼: API 호출 → 주택 목록 이동

### 4.14 입주희망주택 목록 (`/housings`)

**상세기능:**
- 주택 목록 조회
- 정렬 (등록일, 가격)
- 페이징
- 최종목표 주택 표시

**UI 구성요소:**
- AppBar: Back 버튼, "입주희망주택" 타이틀, 추가 버튼
- Select: 정렬 기준
- Card List: 주택 카드
  - Chip: 최종목표 뱃지
  - Typography: 주택명, 주택유형
  - Typography: 가격
  - Typography: 주소
  - Typography: 등록일
- Pagination
- FAB: 주택 추가 버튼

**인터랙션:**
- 카드 클릭: 상세 페이지 이동

## 8. 성능 최적화 방안

### 8.1 코드 스플리팅
- **Route-based**: React.lazy + Suspense
- **Component-based**: 큰 컴포넌트 지연 로딩

### 8.2 이미지 최적화
- **WebP 포맷** 사용
- **Lazy Loading**: Intersection Observer
- **반응형 이미지**: srcset, sizes

### 8.3 API 최적화
- **캐싱**: React Query (staleTime, cacheTime)
- **Prefetching**: 예상 페이지 데이터 미리 로드
- **Debouncing**: 검색 입력 디바운싱

### 8.4 렌더링 최적화
- **React.memo**: 불필요한 리렌더링 방지
- **useMemo, useCallback**: 연산 및 함수 메모이제이션
- **Virtualization**: 긴 리스트는 react-window 사용

### 8.5 번들 최적화
- **Tree Shaking**: 사용하지 않는 코드 제거
- **Compression**: Gzip/Brotli 압축
- **CDN**: 정적 자산 CDN 배포

### 8.6 성능 목표
- **First Contentful Paint**: < 1.5s
- **Time to Interactive**: < 3.5s
- **Largest Contentful Paint**: < 2.5s
- **Cumulative Layout Shift**: < 0.1
- **First Input Delay**: < 100ms
